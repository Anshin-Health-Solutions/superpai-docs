# Cost Management

SuperPAI+ provides real-time cost tracking for all AI model usage across sessions. The cost system monitors token consumption per model, calculates running totals, and provides optimization recommendations.

---

## Real-Time Tracking

Every API call to Claude models is tracked with:

- **Input tokens** --- Tokens sent to the model (your code, context, instructions)
- **Output tokens** --- Tokens generated by the model (responses, code, analysis)
- **Cache tokens** --- Tokens served from prompt caching (reduced cost)
- **Model used** --- Which specific model processed the request
- **Timestamp** --- When the call was made

---

## The /cost Command

```bash
/cost                    # Current session summary
/cost detailed           # Per-call breakdown
/cost today              # Today's total across all sessions
/cost model=genius       # Filter by model alias
/cost --json             # Machine-readable output
```

## Cost Optimization Tips

### 1. Let Adaptive Depth Do Its Job

The CCR (Cost-Conscious Router) automatically routes simple sub-tasks to `simple` (Haiku) and only uses `genius` (Opus) when complexity requires it. Do not override model selection unless necessary.

### 2. Use /quick for Small Tasks

The `/quick` command is optimized for minimal token usage. It skips the planning and review phases that consume additional tokens in full WORKFLOW mode.

### 3. Leverage Prompt Caching

SuperPAI+ automatically benefits from Anthropic's prompt caching. Repeated context (like skill definitions and project structure) is cached, reducing input token costs by up to 90%.

### 4. Be Specific in Requests

Vague requests like "improve the codebase" consume many tokens as the AI explores broadly. Specific requests like "add null check to getUserById in users.ts" are much cheaper.

### 5. Review Before Large Operations

Before running `/audit` (150+ checks) or `/spec` (full specification), ensure you actually need the full scope. Use `--category` or `--scope` flags to limit the work.

### 6. Monitor Daily Costs

Set up a daily cost review habit:

```bash
/cost today        # Check daily spending
/cost --week       # Weekly trend
```

## Cost Alerts

Configure cost alerts in your SuperPAI+ settings:

```json
{
  "cost": {
    "tracking_enabled": true,
    "alert_threshold_session": 10.00,
    "alert_threshold_daily": 50.00,
    "preferred_model": "smart",
    "allow_genius": true,
    "genius_confirmation": false
  }
}
```

| Setting | Default | Description |
|---------|---------|-------------|
| `tracking_enabled` | `true` | Enable cost tracking |
| `alert_threshold_session` | `$10.00` | Warn when session cost exceeds this |
| `alert_threshold_daily` | `$50.00` | Warn when daily cost exceeds this |
| `preferred_model` | `smart` | Default model for standard tasks |
| `allow_genius` | `true` | Allow Opus model usage |
| `genius_confirmation` | `false` | Require confirmation before using Opus |

## Stored Cost Data

Cost data is persisted in the superpai-server SQLite database in the `cost_tracking` table. Historical data is available for trend analysis and budgeting.

```bash
/cost history 30      # Last 30 days of cost data
/cost export csv      # Export cost data to CSV
```
